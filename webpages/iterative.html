Iteration

In solving Karel problems, you will often find that repetition is essential. To see how repetition can be used in the context of a program, consider the problem of Karel filling potholes for a stylized roadway with potholes at every even-numbered avenue:


Our mission is to write a program that instructs Karel to fill all the holes in this road. Since you know that in this problem there are exactly four holes to fill, the control statement that you need is a for loop, which specifies that you want to repeat some operation a predetermined number of times. The structure of the for statement appears complicated primarily because it is actually much more powerful than anything Karel needs. The only version of the for syntax that Karel uses is

    for (var i = 0; i < count; i++) {
        statements to be repeated
    }

where count is an integer indicating the number of repetitions. In english this translates into: repeat the code between the curly brackets count times. Here is an example program that would make karel move forward 10 times:

    for (var i = 0; i < 10; i++) {
        move();
    }
