First Program

Now that we know Karel's four commands, the next step is to string together commands into a runnable program. Here's an example of what that looks like:


firstProgram.w,280,170,first

Run:  

The first important part in this program is the "comments" section. Comments in both Karel and JavaScript begin with the characters /* and end with the characters */. Everything within the comment section (green text) is ignored by the computer and is there to document the program for human readers.

The rest of the program represent the definition of a new function with the name run. the function definition consists of two parts: The first line constitutes the function header and the code between the curly braces is the function body. The special JavaScript word function followed by the word run tells the computer that a new function is being defined that can be called with the name run. For now you can think of the parens after the word run as a bit of syntax that goes with defining functions. The commands that are in the body describe what the computer will execute (in order) when the run function is called. All commands in the body are followed by a semi-colon.

In Karel the run function will always be the entry function that the computer steps through when you execute your program (hit the green play button).
