Conditionals

So far our programs have followed a strictly linear, step-by-step order of operations. Conditional statements make it possible for Karel to examine its world and change its execution pattern accordingly. To control the flow of the program based on an observation Karel can make about the world, we can use the if statement, which ordinarily appears in the following form:

    if (conditional test) {
        statements to be executed only if the condition is true
    }

The conditional test shown in the first line of this pattern must be replaced by one of the tests Karel can perform on its environment. The result of that conditional test is either true or false. If the test is true, Karel executes the statements enclosed in braces; if the test is false, Karel does nothing.

Every condition has a corresponding opposite. For example, you can use either the frontIsClear condition or the frontIsBlocked condition. Here is a list of the test's Karel can perform on the world:


Test Opposite What it checks
frontIsClear() frontIsBlocked() Is there a wall in front of Karel?
leftIsClear() leftIsBlocked() Is there a wall to Karel's left?
rightIsClear() rightIsBlocked() Is there a wall to Karel's right?
beepersPresent() noBeepersPresent() Are there beepers on this corner?
facingNorth() notFacingNorth() Is Karel facing north?
facingEast() notFacingEast() Is Karel facing east?
facingSouth() notFacingSouth() Is Karel facing south?
facingWest() notFacingWest() Is Karel facing west?
